# City Go — Detailed Use Cases (Checklist Level)

## Зачем этот файл

Это **детализация use cases до уровня QA-чеклистов**.

Каждый кейс:
- проверяется как отдельная логическая единица;
- покрывает happy path + edge cases;
- используется для тестирования, разработки и ревью.

---

# UC-020 — Evening Route

## Описание
Пользователь строит маршрут вечером (например, старт в 19:00).

---

## Входные данные

- trip_start: 19:00
- time_budget: 180 min
- city: любой
- interests: не заданы (или общие)

---

## Ожидаемое поведение

- маршрут содержит места, работающие вечером;
- бары/рестораны/вечерние активности имеют приоритет;
- утренние/дневные места не доминируют.

---

## Чек-лист

### Candidate retrieval
- [ ] места, открывающиеся в 18:00+, присутствуют в кандидатах
- [ ] места, закрытые днём, но открытые вечером, НЕ отфильтрованы

### Scoring
- [ ] вечерние категории получают приоритет
- [ ] distance не доминирует над релевантностью

### Route building
- [ ] маршрут логичен по времени
- [ ] нет мест, которые закрываются до момента визита

### Edge cases
- [ ] если все места закрываются рано — есть fallback
- [ ] если нет вечерних мест — пользователь получает explanation

# UC-021 — Closing Soon Prioritization

## Описание
Место закрывается раньше других → должно быть раньше в маршруте.

---

## Входные данные

- trip_start: 14:00
- time_budget: 3h
- place A closes at 17:00
- place B closes at 21:00

---

## Ожидаемое поведение

- place A идёт раньше place B

---

## Чек-лист

### Detection
- [ ] система корректно считает время до закрытия

### Scoring
- [ ] closing soon влияет на приоритет

### Route ordering
- [ ] раннее закрывающиеся места не оказываются в конце

### Edge cases
- [ ] если оба закрываются скоро — учитывается distance
- [ ] если closing неизвестен — fallback логика

# UC-025 — Place Opens Later

## Описание
Место открывается после старта маршрута.

---

## Входные данные

- trip_start: 15:00
- place opens: 18:00
- time_budget: 4h

---

## Ожидаемое поведение

- место НЕ попадает в начало маршрута
- может появиться в конце (если попадает по времени)

---

## Чек-лист

### Candidate pool
- [ ] место НЕ отфильтровано заранее

### Visit time logic
- [ ] visit_time учитывается для позиции

### Route placement
- [ ] место стоит только в допустимой позиции

### Negative cases
- [ ] если не успеваем — место исключается

---

# UC-023 — Waiting Gap Handling

## Описание
Пользователь приходит раньше открытия.

---

## Входные данные

- arrival: 17:30
- opens: 18:00

---

## Ожидаемое поведение

- либо reorder
- либо warning

---

## Чек-лист

### Detection
- [ ] система определяет gap

### Handling
- [ ] small gap (<20 min) допустим
- [ ] большой gap → избегается

### UX
- [ ] пользователь уведомлён

---

# UC-050 — No Places Found

## Описание
Нет мест в радиусе.

---

## Ожидаемое поведение

- не падаем
- возвращаем пустой маршрут
- объясняем причину

---

## Чек-лист

- [ ] нет 500 ошибки
- [ ] explanation содержит причину
- [ ] возможна рекомендация расширить поиск

Все места закрыты.

---

## Чек-лист

# UC-054 — Diversity Collapse

## Описание
Маршрут из одинаковых категорий.

---

## Чек-лист

- [ ] не более 2 одинаковых категорий подряд
