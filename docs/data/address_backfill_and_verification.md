# Address Backfill and Verification

## Цель

Адресный контур должен решать две задачи:

1. Заполнить адреса там, где адреса нет.
2. Перепроверить уже существующие адреса и не ломать ручные/старые данные.

## Как работает базовый режим

Команда:

```bash
python data/scripts/backfill_missing_place_addresses.py \
  --city almaty \
  --limit 500 \
  --sleep 1.1 \
  --apply
```

Что делает:

1. Берёт места выбранного города только с пустым или плейсхолдерным адресом.
2. Для каждого места вызывает reverse geocoding по координатам.
3. Если найден нормальный адрес — записывает его в `places.address`.
4. Ставит служебные поля:
   - `address_source = nominatim`
   - `address_confidence = 0.75`
   - `address_updated_at = now()`
5. Если результат слишком общий или пустой — адрес не записывает.

## Как работает режим перепроверки существующих адресов

Команда:

```bash
python data/scripts/backfill_missing_place_addresses.py \
  --city almaty \
  --limit 500 \
  --sleep 1.1 \
  --apply \
  --verify-existing
```

Что меняется:

1. В выборку попадают не только места без адреса, но и места с уже заполненным адресом.
2. Скрипт получает новый адрес по координатам.
3. Сравнивает текущий адрес и новый адрес.

## Правила сравнения адресов

Сравнение не строковое один-в-один, потому что один и тот же адрес может выглядеть по-разному:

```text
улица Абая, 10
Абая 10, Алматы
проспект Абая, дом 10
```

Скрипт нормализует адреса:

- приводит к нижнему регистру;
- убирает пунктуацию;
- убирает служебные слова вроде `улица`, `проспект`, `дом`;
- сравнивает значимые токены;
- отдельно учитывает номера домов.

## Если адреса совпали

Если текущий адрес и найденный адрес совпадают достаточно уверенно:

1. Текущий адрес не меняется.
2. Место считается подтверждённым по адресу.
3. Обновляются служебные поля:
   - `address_source`, если он был пустой;
   - `address_confidence`, минимум до `0.75`;
   - `address_updated_at`.

Статистика прогона увеличивает:

```text
verified_existing
```

## Если адреса отличаются

Если текущий адрес и найденный адрес заметно отличаются:

1. Текущий адрес не перезаписывается.
2. Место отправляется на ревью.
3. Ставятся поля:
   - `verification_status = needs_recheck`
   - `verification_source = nominatim`
   - `verification_method = address_conflict`
   - `verification_comment = Current + Candidate`
   - `needs_recheck_at = now()`

Статистика прогона увеличивает:

```text
sent_to_review
```

## Почему нельзя сразу перезаписывать отличающиеся адреса

Reverse geocoding может вернуть:

- адрес соседнего здания;
- адрес торгового центра вместо конкретного заведения;
- адрес ближайшей дороги;
- район вместо точного объекта;
- технически правильный, но продуктово менее полезный адрес.

Поэтому при конфликте нужна ручная проверка.

## Что смотреть в результате прогона

Основные поля статистики:

```text
checked — сколько мест проверено
updated — сколько пустых адресов заполнено
verified_existing — сколько существующих адресов подтверждено
sent_to_review — сколько конфликтов отправлено на ревью
skipped_no_coordinates — сколько мест без координат
skipped_generic_result — сколько результатов слишком общие
errors — сетевые/технические ошибки
```

## Рекомендуемый порядок запуска по городу

### Шаг 1. Сначала добрать пустые адреса

```bash
python data/scripts/backfill_missing_place_addresses.py \
  --city almaty \
  --limit 500 \
  --sleep 1.1 \
  --apply
```

### Шаг 2. Потом перепроверить существующие адреса

```bash
python data/scripts/backfill_missing_place_addresses.py \
  --city almaty \
  --limit 500 \
  --sleep 1.1 \
  --apply \
  --verify-existing
```

### Шаг 3. Разобрать ревью

В админке нужно отфильтровать места:

```text
verification_status = needs_recheck
verification_method = address_conflict
```

И вручную принять решение:

- оставить текущий адрес;
- заменить адрес на предложенный;
- поправить адрес вручную;
- отметить место как проблемное.

## Ограничения

1. Nominatim нельзя дергать агрессивно.
2. Безопасная задержка: `--sleep 1.1`.
3. Большой город нужно обрабатывать батчами.
4. Один батч лучше держать в пределах 300–500 мест.
5. Конфликты нельзя применять автоматически.

## Следующее улучшение

Нужно вывести `address_conflict` в админскую очередь верификации с фильтрами:

- город;
- категория;
- причина ревью;
- текущий адрес;
- предложенный адрес;
- кнопки: оставить / заменить / исправить вручную.
