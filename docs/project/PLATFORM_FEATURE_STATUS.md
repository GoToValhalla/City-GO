# City Go — статус функционала платформы

Последнее обновление: 2026-06-18

Документ фиксирует текущий статус функционала платформы и админки. Используется как рабочий чек-лист готовности продукта.

## Обозначения статусов

- ✅ Реализовано
- 🟡 Частично реализовано
- ⚠️ Реализовано технически, но требует доработки
- ❌ Не реализовано

## Правила обновления

1. После каждой реализованной функции обновлять статус в этом документе.
2. Статус ✅ ставить только после проверки в интерфейсе/API и отсутствия критичных ошибок.
3. Для спорных или неполных функций использовать 🟡 или ⚠️.
4. Если функциональность есть в коде, но не готова для пользователя, использовать ⚠️.
5. При добавлении новых функций дополнять этот чек-лист.

## Стратегический порядок разработки

### Этап 1. Данные и качество каталога

Цель: добиться высокого доверия к данным.

Приоритет:
- адреса;
- координаты;
- фото;
- описания;
- режимы работы;
- очистка мусорных POI;
- корректные категории и теги;
- confidence model;
- freshness model;
- quality score места;
- quality score города.

### Этап 2. Маршрутный движок

Цель: сделать маршруты главным продуктом.

Приоритет:
- исправление багов маршрутов;
- защита от пустых маршрутов;
- защита от дублей;
- Route Constructor;
- пользовательские якоря;
- ручное редактирование маршрута;
- drag and drop;
- пересчет маршрута;
- альтернативные точки;
- предупреждения.

### Этап 3. Карта и UX

Цель: сделать продукт удобным для ежедневного использования.

Приоритет:
- полноценная карта;
- отображение всех точек;
- фильтрация;
- карточки мест;
- открыто/закрыто;
- карта маршрута;
- активный маршрут;
- завершенный маршрут;
- просмотр места из карты.

### Этап 4. Пользовательский слой

Цель: накопление пользовательских данных и удержание.

Приоритет:
- регистрация;
- авторизация;
- профиль;
- интересы;
- избранное;
- хочу посетить;
- посетил;
- оценки;
- отзывы;
- история маршрутов.

### Этап 5. AI, Telegram и монетизация

Цель: создание уникальности продукта.

Приоритет:
- recommendation engine;
- user memory;
- поведенческий профиль;
- explainable recommendations;
- Telegram Bot нового поколения;
- монетизация;
- донаты;
- премиум функции;
- ачивки;
- квесты.

### Что не является приоритетом сейчас

До завершения первых трех этапов не рекомендуется тратить основные ресурсы на:

- ачивки;
- квесты;
- донаты;
- комментарии;
- отзывы;
- социальные функции;
- сложную AI-персонализацию.

Ключевой порядок развития продукта:

Этап 1 → Данные
Этап 2 → Маршруты
Этап 3 → Карта
Этап 4 → Пользователи
Этап 5 → AI / Telegram / Монетизация
