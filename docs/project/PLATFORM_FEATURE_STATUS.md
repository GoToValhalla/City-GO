# City Go — статус функционала платформы

Последнее обновление: 2026-06-18

Документ фиксирует текущий статус функционала платформы и админки. Используется как рабочий чек-лист готовности продукта.

## Обозначения статусов

- ✅ Реализовано
- 🟡 Частично реализовано
- ⚠️ Реализовано технически, но требует доработки
- ❌ Не реализовано

## Правила обновления

1. После каждой реализованной функции обновлять статус в этом документе.
2. Статус ✅ ставить только после проверки в интерфейсе/API и отсутствия критичных ошибок.
3. Для спорных или неполных функций использовать 🟡 или ⚠️.
4. Если функциональность есть в коде, но не готова для пользователя, использовать ⚠️.
5. При добавлении новых функций дополнять этот чек-лист.

# Готовность крупных блоков

| Блок | Текущая оценка готовности | Комментарий |
|---|---:|---|
| Движок маршрутов | 65–70% | Сильный технический слой: scoring, фильтры, interests mapping, correction, trace/debug. Требуется полноценный пользовательский UX. |
| Каталог мест | 50–60% | База мест и карточки есть, но качество данных неполное: фото, адреса, описания, часы работы. |
| Админка | 35–40% | Есть важные операционные экраны, но не хватает авторизации, полноценного управления, фича-тоглов и UX. |
| Телеграм-бот | 20–25% | Бот отстает от основного продукта и требует переосмысления пользовательских сценариев. |
| Пользователи и профили | 5–10% | Регистрация, авторизация, профиль, списки и избранное не готовы. |
| Социальные функции | 0% | Комментарии, оценки и шаринг не готовы. |
| AI-персонализация | 15–20% | Есть retrieval/orchestration слой и учет интересов, но нет user memory и поведенческого профиля. |
| Монетизация | 0% | Донат и оплаты не реализованы. |
| Карта | 15–20% | Требуется полноценный интерактивный UX карты, карточки, фильтры и состояния мест. |
| Геймификация | 0% | Ачивки и квесты не реализованы. |

# Основной вывод

На текущем этапе сильнее всего развиты внутренние технические слои: маршрутный движок, каталог мест, импорт/обогащение данных и контроль качества данных.

Самые большие пробелы находятся в пользовательском слое: авторизация, профили, карта, избранное, отзывы, персонализация, социальные функции, монетизация и полноценный UX маршрутов.
